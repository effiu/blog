HikariCP

Posted on 2020-05-04 22:17 Edited on 2020-05-05 20:34 In Java Views: Valine: Reading time ≈ 20 mins.

HikarCP是快速、简单、可靠的、非常轻量的JDBC连接池。

1. HikariConfig

1. 基本配置项

  • dataSourceClassName:JDBC驱动程序提供的DataSource类的名称。如果使用jdbcUrl进行基于老式DriverManager的JDBC驱动程序配置，则不需要此属性。见源码:com.zaxxer.hikari.pool.PoolBase#initializeDataSource中

  • jdbcUrl:HikariCP会基于DriverManager进行DataSource进行数据源的配置，基于数据源的配置优于基于jdbcUrl的配置，但是实际差异不大。

  • username:设置从基础驱动程序获取连接时使用的身份验证的用户名。

  • password:设置从基础驱动程序获取链接时使用的身份验证的密码。

  • isAutoCommit:控制从连接池返回的连接的自动提交行为，默认为true

  • connectionTimeout:控制客户端等待来自池的连接的最大毫秒数，最小值为250ms，默认值为Integer.MAX_VALU。

  • idleTimeout:控制池中处于空闲状态的连接的最长时间。仅仅当minimumIdle小于maximumPoolSize时才会起作用，否则当达到maximumPoolSize后将不会销毁。若值为0，则表示永远不会销毁，最小值为10s，默认值为10m。

  • maxLifeTime:此属性控制池中连接的最大生存期。使用中的连接永远不会停止使用，只有在关闭后才将其删除。0表示无限寿命(会遵守idleTime设置)，默认30m。

  • connectionTestQuery:若支持JDBC4，则不推荐设置该属性。是针对不支持JDBC4的驱动程序在从池中提供连接之前执行的查询，以验证与数据库的连接仍然有效。

    JDBC4以后，connection提供了isValid方法，不再需要改配置检查连接,见com.zaxxer.hikari.pool.PoolBase#isConnectionAlive。

  • minIdle:连接池中的最小空闲数，若空闲连接下降到该值以下，且池中总连接数小于maxPoolSize,则HikariCP将最大努力快速有效地添加连接。为了获得最佳性能和对峰值需求的响应能力,不建议设置此值。默认等于maxPoolSize。

  • maxPoolSize:控制允许池内连接数达到的最大值，包括空闲中和使用中的连接数。当池内连接数达到最大且空闲连接为0时，getConnection()将阻塞connectioinTimeout秒。

  • poolName:表示连接池的用户定义名称，主要用于在日志或者JMX管理控制台中，以识别池信息，默认自动生成，poolName = "HikariPool-" + POOL_NUMBER.getAndIncrement();。

  • connectionInitSql:设置一个SQL语句，每次创建连接后执行，然后把连接加入到连接池中。

2. 配置方式

  1. new HikariCP()，其是通过System.setProperty("hikaricp.configurationFile", filePath)获取配置文件路径，将配置文件加载为Properties，然后通过反射将文件内的配置信息初始化。
  2. new HikariConfig(Properties properties)，直接通过反射将properties内的配置信息初始化连接池配置。
  3. new HikariConfig(String propertyFileName)与1类似。
