LockSupport工具与Condition接口

Posted on 2019-05-18 17:45 Edited on 2019-09-02 00:54 In concurrent Views: Valine: Reading time ≈ 5 mins.
5. LockSupport工具

当我们需要阻塞或者唤醒一个线程的时候，需要使用java.util.concurrent.locks.LockSupport来完成。java.util.concurrent.locks.LockSupport是构建同步组件的基础工具，其内部定义了一组公共静态方法，提供了基本的线程阻塞和唤醒功能。

java.util.concurrent.locks.LockSupport定义了一组park*开头的方法用来阻塞当前线程，unpark(Thread thread)方法来唤醒一个被阻塞线程，方法描述见下表：

方法名称 描述
void park() 阻塞当前线程，需要使用unpark方法或者当前线程被中断，才能够从park()方法返回
void parkNanos(long nanos) 阻塞当前线程，在park()基础上增加了超时返回
void parkUntil(long deadline) 阻塞当前线程，直到deadline时间(1970年开始到deadline的时间毫秒数)
void unpark(Thread thread) 唤醒处于阻塞状态的线程

Java 6中增加了以下3个方法，用来展示阻塞对象blocker，代码片段和线程dump，便于开发定位问题

  • void park(Object blocker);
  • void parkNanos(Object blocker, long nanos);
  • void parkUntil(Object blocker, long deadline);

6. Condition接口

任意一个Java对象都有一组监视器方法(在Java.lang.Object上)，主要包括wait()、wait(long timeout)、notify()和notifyAll()方法，与synchronized配合，可以实现等待/通知模式。

Condition接口也有监视器方法，与Lock配合可以实现等待/通知模式。以下是对比Object的监视器方法和Condition接口：

对比项 Object Monitor Methods Condition
前置条件 获取对象的锁 调用Lock.lock()获取锁，Lock.newCondition()获取Condition对象
调用方式 object.wait condition.await()
等待队列个数 一个 多个
当前线程释放锁并进入等待状态 支持 支持
当前线程释放锁进入等待状态且不响应中断 不支持 支持
当前线程释放锁并进入超时等待状态 支持 支持
当前线程释放锁进入超时等待状态到将来某个时间 不支持 支持
唤醒等待队列中的一个线程 支持 支持
唤醒等待队列中的全部线程 支持 支持

2. Condition实现分析

ConditionObject是AbstractQueuedSynchronizer内部类，Condition操作需要获取相关联的锁，每个Condition对象都包含着一个等待队列，该队列是等待/通知功能的关键

  1. 等待队列

等待队列是一个FIFO队列，队列中每个节点都包含一个线程引用，该线程就是在Condition上等待的线程，Condition.await()方法会使该线程释放锁、构造成节点加入到等待队列中进入等待状态。实际上，同步队列和等待队列中节点类型都是同步器的静态内部类AbstractQueuedSynchronizer.Node。

一个Condition包含一个等待队列，Condition包含首节点和尾节点，Condition.await()会将线程构造成节点放入到尾节点，进入等待状态。新增节点只需要，将尾节点的nextWaiter指向新增节点，更新Condition的lastWaiter即可。调用await()的线程必定持有锁，同时也保证了线程安全以及等待队列的线程安全

Lock可以有多个Condition，所以可以存在多个等待队列，即拥有一个同步队列和多个等待队列。

  1. 等待

Condition.await()会使线程进入等待队列并释放锁、线程状态变为等待状态，当从await()返回时，线程一定重新获得了锁。当调用await()方法时，线程从同步队列的首节点，进入到等待队列的尾节点。如下代码所示：

  1. 通知

Condition.signal()方法，唤醒在等待队列中时间最长的节点(首节点)，唤醒之前会将节点移动到同步队列中。前置条件是当前线程获取到锁。

enq(Node node)方法将等待队列的头节点线程安全地移动到同步队列，再使用LockSupport.unpark(Thread thread)唤醒该节点的线程。被唤醒的线程从await()的for循环中退出，调用同步器的acquiredQueued()方法加入到获取同步状态的竞争中。

signalAll()方法，相当于将等待队列中的所有线程加入到同步队列中，并唤醒所有节点的线程。
